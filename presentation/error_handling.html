Error Handling

Result Wrapper Pattern
sealed class Result<out T> { data class Success<T>( val data: T ) : Result<T>() data class Error( val exception: String ) : Result<Nothing>() object Loading : Result<Nothing>() }
Error Scenarios Handled
🔴 No Internet Connection
Network unavailable or device offline
User sees: "Check your internet connection"
🔴 Invalid Coordinates
Latitude/longitude outside valid ranges
User sees: "Invalid location coordinates"
🟠 API Timeout
Request exceeds 30-second timeout
User sees: "Request timed out, try again"
🟠 JSON Parsing Error
Malformed API response or schema mismatch
User sees: "Failed to parse weather data"
🔵 Retry Functionality
Error screen displays retry button
User can retry without restarting app
